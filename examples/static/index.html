健康检查

向 GET /health 发送请求；该路径在 AuthMiddleware 白名单内， 会经过 AccessLog → RateLimiter → CORS → Auth(放行) → Gzip， 最终由 handleHealth 返回固定 JSON {"status":"ok"}。常用于探活。

尚未请求

Prometheus 指标

向 GET /metrics 拉取由 ServerMetrics 单例聚合的进程内指标 （请求总数、状态码分布、字节数等），输出文本格式由 toPrometheus() 生成， 可直接被 Prometheus / VictoriaMetrics 抓取。

尚未请求

受保护 API（鉴权）

向 GET /api/users 发起请求；AuthMiddleware 检查 Authorization: Bearer ... 或 X-API-Key，匹配 demo-token-2024 / api-key-001 任一即放行；否则短路返回 403。

尚未请求

CORS 预检

发送 OPTIONS /api/users 并附上 Origin / Access-Control-Request-Method；CorsMiddleware 识别为预检后短路， 直接返回 204 + 四个 Access-Control-* 头，不会进入路由层。

尚未请求

静态文件 / 下载

向 GET /files/<name> 拉取文件；前缀路由命中 handleStatic → StaticFileHandler::serve： 自动 MIME 推断、生成 ETag、Linux 走 sendfile() 零拷贝 （macOS 退化为 read+send）。

尚未请求

令牌桶限流

并发发起 N 次 GET /health；RateLimiter 按 per-IP 令牌桶 （容量 200、补充速率 100/s）限流，超出部分立即被中间件短路为 429 Too Many Requests。 可直观看到 200 / 429 的分布。

尚未请求

ETag / 304 协商缓存

第一次 GET /files/readme.txt 取回 ETag； 第二次带 If-None-Match 重发，StaticFileHandler 比对后命中缓存， 返回 304 Not Modified 且无响应体，节省带宽。

尚未请求

Range 分段下载

带 Range: bytes=0-31 请求 /files/readme.txt； 处理器解析区间后返回 206 Partial Content + Content-Range，仅传输指定字节区间，是断点续传 / 流媒体的基础。

尚未请求

Gzip 响应压缩

浏览器默认携带 Accept-Encoding: gzip；GzipMiddleware 在响应阶段判断响应体 ≥ 256 字节且 Content-Type 可压缩时，原地替换 body 并写入 Content-Encoding: gzip。浏览器透明解压，可在 DevTools 看到原始压缩字节数。

尚未请求

文件管理器（实时）

通过 GET /api/files 列出 ./files 目录； 上传走 POST /api/files（multipart/form-data，由 HttpRequest::parseMultipart 解析后落盘）； 删除走 DELETE /api/files/<name>。每次操作后自动刷新列表， 并每 5 秒轮询一次以反映外部变更。
